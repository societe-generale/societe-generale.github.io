Some of the projects you could work on...

1. Revamping of 4 legacy applications into a single, state-of-the-art micro-services platform.
Thousands of internal users, worldwide.

Domain: Investment Banking / Structured Finance

Description: To accommodate the planned growth of the business, it has been agreed that current “multiple monoliths” legacy architecture would not allow us to match the speed at which business teams want to move. Therefore, it has been decided to move towards a micro-services architecture in the back-end (following DDD and CQRS patterns), and an AngularJs front-end. New code is directly designed and written following these principles, and whenever we have the opportunity, we migrate the legacy features to the new architecture. This architecture, along with a radical change in mindset (“a feature is complete when the automated unit/integration tests are validated and passing”) will allow us to deliver in production at the end of every sprint and avoid big bang releases. Eventually, we anticipate that the notion of “application” will not be relevant anymore : there will be one UI plugged on a set of REST services.

Technologies: Java8, Sprint Boot, Spring Cloud, Oracle, Axon framework

Practices and Tools: TDD, BDD, Continuous Delivery, GitHub Enterprise, Clean Code

2. MyALD, a digital tool accessible via a web portal and as a mobile app, designed to simplify the ALD drivers’ and fleet managers’ journey, deployed in 28 countries in Europe, Asia, LatAM

Domain: Car Leasing

Description: My ALD is an omnichannel digital tool, accessible via a web portal and mobile app for fleet managers and drivers that encompasses the main interactions customers have with ALD Automotive during their contract. The tool digitalizes the key moments in their journey and enriches their experience when using the ALD leasing services and vehicle: configure and order vehicles, request services, update mileage, access contract details in real-time and fleet reports, documents, geo-locate service partners. The mobile app answers to the specific driver needs while they are on the go, thus enriching their ALD experience with a complete mobility. Both products are undergoing constant developments in terms of functionalities and user interface.

Technologies: Share point 2010/2016,ADFS 2.0 WCF 3.0, Dot net Framework 3.5, SQL Server, Bootstrap 3.3.2, JQuery, Logi Info

Practices and Tools: Agile, Continuous Delivery, TFS, Altiris, Visual Studio 2010

4. eCredApp

Domain: Private Banking / Credit Workflow

Description: eCredApp is a platform independent digital tool, accessible via a web portal and mobile app. It is an in-house application that standardizes & structures our credit application process in order to facilitate interactions, seamlessly removing all manual process delays. The application integrates and automates existing workflow from initiation to completion of credit approval ensuring transparency & monitoring of the process. The mobile app can be accessed by the Private banker while they are on the go, thus enriching their eCredApp experience. Also, this application will be extended directly to private banking clients i.e. HNIs, UHNWs which will give them the flexibility to raise credit request and submit it to their private banker. Application is being developed using latest technologies, incorporates the SOA principles and follows advanced continuous delivery practices.

Technologies: Linux RHEL, Java8, AngularJs, REST APIs, Apache Tomcat 8, Activiti, Oracle 12c

Practices and Tools: Agile, TDD, BDD, Continuous Delivery, DevOps. CI with GitHub, Jenkins, Maven, Sonar, Nexus.
